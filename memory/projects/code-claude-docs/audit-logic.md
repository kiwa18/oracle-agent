# 一致性审计逻辑

## 核心理念

CODE 不是白痴核验。不做格式检查。做**多指标指向性判断**。

## 审计方法

### 1. 方向一致性

```
价格方向 × 量能方向 × 指标方向 = 指向性

指向一致 → ✅ 可信
指向打架 → ❌ 打回
```

### 2. 矛盾检测

| 矛盾类型 | 示例 | 严重度 |
|------|------|:--:|
| 价量背离 | 价格↑ + 缩量 | 高 |
| 指标打架 | MACD金叉 + RSI超卖 | 中 |
| 趋势矛盾 | K线多头 + MACD死叉 | 中 |
| 数据自洽 | 总和不对、数量不匹配 | 高 |

### 3. 来源审计

```
每条数据：
  ✅ 有拉取时间戳
  ✅ 有数据源标注
  ✅ 缺失字段 + 原因 + 替代方案
  ❌ 缺任一项 → 打回
```

## 审计流程

```
收到结果
  ├── ① 审计元数据（来源/时间戳/缺失/替代）→ 不完整就打回
  ├── ② 审计自洽性（数量/范围/正负）→ 异常就打回
  ├── ③ 审计一致性（价量指标方向）→ 打架就打回
  └── ④ 全部通过 → 整理推微信
```

## 打回格式

```
"结果有问题，打回重跑：
1. 时间戳缺失——请标注每条数据的拉取时间
2. 价量矛盾——价格涨了4%但成交量萎缩，方向不一致
3. 融资融券字段为空——说明原因或换备源"
```

## 通过格式

```
"[润色后的中文总结]

来源：mootdx + 腾讯K线 | 拉取时间：2026-06-14 15:30"
```
