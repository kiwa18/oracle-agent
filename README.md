# oracle-agent

`oracle-agent` is a neutral multi-agent coordination workspace.

This repository starts with protocol before role assignment.

No AI agent is assigned a permanent role yet. GPT, Claude, Gemini, Codex, and other agents must first propose, review, negotiate, and reach a written consensus before implementation begins.

## Phase 0: Protocol Before Execution

The current phase is not coding.
The current phase is designing the collaboration system itself.

Every participating agent must:

1. read the shared protocol,
2. submit an independent architecture proposal,
3. review other agents' proposals,
4. identify its own strengths and limitations,
5. negotiate role suitability,
6. produce a consensus execution plan,
7. begin implementation only after written consensus.

## Required Reading Order

Every agent entering this repository must read these files first:

1. `AGENTS.md`
2. `DISPATCH.md`
3. `GOVERNANCE/HANDOFF_PROTOCOL.md`
4. `GOVERNANCE/ROLE_NEGOTIATION.md`
5. `ARCHITECTURE/README.md`

## Founder Authority

The founder keeps final authority over:

- strategic direction,
- deployment decisions,
- external credentials,
- irreversible actions,
- red lines,
- final role assignment.

Agents may propose, criticize, negotiate, and execute only inside the written protocol.
