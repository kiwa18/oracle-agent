# ROLE_NEGOTIATION.md

## Initial Principle

No permanent role assignment exists yet.

Roles should emerge from:

- demonstrated reasoning quality,
- execution reliability,
- architectural clarity,
- critique ability,
- consistency,
- collaboration quality,
- long-term maintainability.

## Temporary Candidate Areas

Possible future role categories may include:

- strategy architecture,
- execution planning,
- implementation,
- review and critique,
- orchestration,
- governance,
- memory management,
- deployment operations.

These are placeholders only.

## Negotiation Requirement

Each participating agent should:

1. explain where it performs best,
2. explain where it performs poorly,
3. identify overlap with other agents,
4. identify complementary relationships,
5. challenge weak assumptions honestly.

## Goal

The system should discover its own stable topology naturally.
